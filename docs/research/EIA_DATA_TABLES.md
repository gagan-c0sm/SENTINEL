# SENTINEL — EIA API Data Tables: What We Use and Why

> Generated by querying the live EIA v2 API catalog at `https://api.eia.gov/v2/`

---

## How the EIA API Works

- **Base URL:** `https://api.eia.gov/v2/`
- **Authentication:** API key as query parameter (`?api_key=YOUR_KEY`)
- **Pagination:** Max **5,000 rows per request**; use `offset` + `length` to page through
- **Format:** JSON responses with `response.data[]` array
- **Source form for hourly data:** EIA-930 (Electric System Operating Data)

---

## The 6 Tables We Use (and Why)

### TABLE 1: Hourly Demand, Forecast, Generation & Interchange

| Property | Value |
|---|---|
| **API Route** | `electricity/rto/region-data/data/` |
| **Source Form** | EIA-930 |
| **Frequency** | Hourly |
| **History** | July 2015 → present (~10.5 years) |
| **Facets** | `respondent` (BA code), `type` (D/DF/NG/TI) |
| **Data columns** | `value` (MWh) |
| **Coverage** | All 65 Balancing Authorities |

**What's in it:** Four data types in one table, filtered by `type` facet:
- `D` = Actual Demand (MWh)
- `DF` = Day-ahead Demand Forecast (MWh)
- `NG` = Net Generation (MWh)  
- `TI` = Total Interchange (MWh, positive = exports)

**Why we need it:**
- `D` (Demand) → **This is our primary prediction target.** Every model we build predicts future demand.
- `DF` (Forecast) → **Benchmark to beat.** If our models can't outperform EIA's own forecast, we haven't added value.
- `NG` (Net Generation) → Tells us if supply is keeping up with demand. The gap between D and NG signals stress.
- `TI` (Interchange) → Shows power flowing in/out of each BA. A BA suddenly importing more power means local supply is strained.

**Example API call:**
```
GET /v2/electricity/rto/region-data/data/
    ?api_key=YOUR_KEY
    &frequency=hourly
    &data[0]=value
    &facets[respondent][]=ERCO       ← ERCOT (Texas)
    &facets[type][]=D                ← Actual demand
    &start=2021-01-01T00
    &end=2026-03-23T00
    &sort[0][column]=period
    &sort[0][direction]=asc
    &offset=0
    &length=5000
```

### TABLE 2: Hourly Generation by Energy Source (Fuel Type)

| Property | Value |
|---|---|
| **API Route** | `electricity/rto/fuel-type-data/data/` |
| **Frequency** | Hourly |
| **History** | July 2018 → present (~7.5 years) |
| **Facets** | `respondent` (BA), `fueltype` |
| **Data columns** | `value` (MWh) |

**Fuel type facets include:**
- `NG` = Natural Gas
- `SUN` = Solar
- `WND` = Wind
- `WAT` = Hydro
- `NUC` = Nuclear
- `COL` = Coal
- `OTH` = Other
- `OIL` = Petroleum

**Why we need it:**
This is the **foundation for cascading effects analysis**. It tells you *what* each BA is burning to generate power.

- If ERCOT is 48% natural gas → it's vulnerable to gas price shocks
- If CAISO is 25% solar → it's vulnerable to cloudy days
- When global oil prices spike and a BA is 5% petroleum, the direct impact is small
- But if oil triggers gas price correlation (substitution effect) and that BA is 48% gas → cascading impact is large

**This table answers:** "Which BAs depend on which fuels, and by how much?"

### TABLE 3: Hourly Interchange by Neighboring BA

| Property | Value |
|---|---|
| **API Route** | `electricity/rto/interchange-data/data/` |
| **Frequency** | Hourly |
| **History** | July 2015 → present |
| **Facets** | `respondent` (source BA), `fromba`/`toba` (neighbor BA) |
| **Data columns** | `value` (MWh) |

**Why we need it:**
This builds the **network graph** for cross-BA risk propagation (Depth Level 5 in our research).

- **Nodes** = Balancing Authorities
- **Edges** = Hourly power flow between them
- When BA-A is stressed, does it draw more from BA-B? Does BA-B then draw from BA-C?
- This is the data behind the "stress propagation" analysis — genuinely underexplored in literature

**This table answers:** "How does grid stress in one region cascade to neighbors?"

### TABLE 4: Natural Gas Spot Prices (Henry Hub)

| Property | Value |
|---|---|
| **API Route** | `natural-gas/pri/fut/data/` (futures) and `natural-gas/pri/sum/data/` (summary) |
| **Frequency** | Daily (weekdays) / Monthly |
| **History** | 1990s → present |
| **Key series** | Henry Hub Natural Gas Spot Price |
| **Data columns** | `value` ($/MMBtu) |

**Why we need it:**
Natural gas is the **#1 fuel for marginal electricity generation** in the U.S. (~40% of total generation). Gas prices directly affect:

1. **Generation cost** → Higher gas prices = more expensive electricity
2. **Merit order** → When gas is expensive, coal/renewables get dispatched more
3. **Cascading trigger** → Oil price shocks correlate with gas prices (0.6–0.8 correlation)

**This table answers:** "What does it cost to generate power from gas right now, and is that changing?"

---

### TABLE 5: Petroleum (Crude Oil) Spot Prices

| Property | Value |
|---|---|
| **API Route** | `petroleum/pri/spt/data/` |
| **Frequency** | Daily (weekdays) |
| **History** | 1986 → present |
| **Key series** | WTI Cushing spot price, Brent Europe spot price |
| **Data columns** | `value` ($/barrel) |

**Why we need it:**
Oil isn't a major fuel for U.S. electricity (~0.5% of generation), but it's the **trigger for cascading effects**:

```
Geopolitical event (e.g., Strait of Hormuz)
    → Oil price spike
        → Gas price correlation (substitution effect)
            → Gas generation cost increase
                → Merit order shift in gas-dependent BAs
                    → Potential supply-demand gap
```

Oil is also the **most news-sensitive** commodity — NLP picks up oil disruption signals faster than gas.

**This table answers:** "Is there a global energy price shock that could cascade into the U.S. grid?"

---

### TABLE 6: Nuclear Outages

| Property | Value |
|---|---|
| **API Route** | `nuclear-outages/us-nuclear-outages/data/` |
| **Source** | NRC Power Reactor Status Report |
| **Frequency** | Daily |
| **History** | Several years |
| **Data columns** | Capacity, outage capacity, percent outage |

**Sub-routes available:**
- `us-nuclear-outages` → National aggregate
- `facility-nuclear-outages` → Per plant
- `generator-nuclear-outages` → Per generator

**Why we need it:**
Nuclear provides **~20% of U.S. electricity** as baseload (always-on). When a nuclear plant goes offline:

- That's 1,000–1,400 MW suddenly missing
- The BA must replace it with gas/coal/imports
- If this happens during peak demand + high gas prices → triple stress event

**This table answers:** "Is there sudden baseload capacity loss that compounds other risks?"

## What We Do NOT Use (and Why)

| EIA Route | Why We Skip It |
|---|---|
| `electricity/retail-sales` | Monthly retail prices — too coarse for hourly prediction |
| `electricity/electric-power-operational-data` | Monthly/annual — we use the hourly RTO data instead |
| `electricity/facility-fuel` | Per-plant data — too granular for national study |
| `electricity/operating-generator-capacity` | Static capacity data — useful reference but not a time-series |
| `coal` | Coal is declining (<20% of generation); captured in fuel-type data |
| `crude-oil-imports` | Import logistics — too far removed from electricity |
| `seds` | Annual state-level — too coarse for hourly analysis |
| `steo` | Short-term outlook projections — useful for validation but not core data |
| `international` | Global data — we focus on U.S. grid |
| `co2-emissions` | Deprecated route |
